## Architektura systemu (architecture)

Opis architektury wyjaśnia **jak system jest zbudowany technicznie**, np.:
- jakie moduły/warstwy istnieją (API, Application, Domain, Infrastructure),
- jak komunikują się między sobą,
- jakie zewnętrzne systemy są używane (baza danych, message broker, cache).

Typowy dokument architektoniczny w tym katalogu zawiera:
- **widok warstwowy lub modułowy systemu**,
- **diagram przepływu danych lub żądań**,
- **mapowanie decyzji architektonicznych (ADR)** na konkretne elementy rozwiązania.

Tutaj łączysz wymagania biznesowe z konkretną strukturą techniczną systemu. Dokumenty z tego katalogu pokazują, jak przypadki użycia i model domenowy przekładają się na układ warstw, zależności i komponentów w kodzie.
