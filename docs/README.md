## Decyzje architektoniczne
ADR (Architecture Decision Record) to krótki dokument opisujący ważną decyzję architektoniczną w projekcie, np.:
- dlaczego wybrano Redis zamiast PostgreSQL do cache,
- dlaczego użyto gRPC zamiast REST,
- dlaczego wybrano RabbitMQ zamiast Kafka.

Typowy ADR zawiera:
- kontekst problemu,
- podjętą decyzję,
- konsekwencje tej decyzji (pozytywne i negatywne).

Szczegółowe ADR-y dla tego projektu znajdują się w katalogu:

- docs/decisions
